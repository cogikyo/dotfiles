# Simulation substrate

Owned simulations are the channel's most honest generative source: Go programs computing the actual systems the videos discuss (cellular automata, memetic drift, selection, entropy, phase spaces) render as deterministic, zero-marginal-cost b-roll and explorables.
The genome permits generative imagery only where artifacts are diegetic; owned simulation is that rule's native substrate, and cloud video generation competes only where computed pixels cannot go.

## Properties

- Deterministic: seeds and parameters live in the project; re-renders are honest in a way cloud sampling never is.
- Diegetic: the pixels are the argument, never an illustration of it; a Galton board on screen is a computed Galton board.
- Owned: house style emerges from code, evolves with the genome token file, and compounds across videos.
- Cheap where cloud is expensive: iteration costs compute, not credits, and every accepted frame is reproducible.

## Render paths

- Simulations render through the browser frame-step contract (canvas or WebGL seeking on time) or by direct frame emission from Go; both terminate in the same alpha-capable clip artifact.
- Ambient title-card fields (cellular automata, reaction-diffusion) come from this substrate, satisfying the genome's ambient-motion exception with content that is real, not decorative.

## Explorable siblings

- A scene that seeks on time for rendering is one flag away from a shippable interactive page where the reader drives the simulation.
- The same scene code yields a video segment and an explorable artifact: one genome, two phenotypes, and the deepest cheap instance of the channel's thesis.

## Open questions

- First simulation: the one the first video's argument actually needs, never a demo for its own sake.
- Library shape: per-video scenes first; a shared scene library only after repetition proves the seams.
- Whether direct-from-Go frame emission earns its place before the browser path covers everything video one needs.
